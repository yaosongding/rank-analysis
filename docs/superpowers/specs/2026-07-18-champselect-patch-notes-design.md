# 选人 AI 分析接入国服版本英雄改动 — 设计

日期：2026-07-18
状态：已口头批准（方案 A + 仅选人 prompt），本文档为实现依据

## 背景

选人期 AI 分析 prompt（`champSelect.ts`）刚修完分路幻觉（本局分路透传 + 敌我前缀纪律）。
多模型实测显示残留问题：qwen-max 无视"禁止编造技能机制"纪律，会编造英雄技能数值。
仓库已维护国服更新公告的英雄改动数据（`patch-notes-data` workflow → Rust
`get_champion_patch_note` 命令 → 前端 `services/patchNotes.ts`，`PatchNoteBadge.vue` 在用）。

## 目标

1. 把本局双方英雄的本版本官方改动喂进选人 prompt，作为阵容/对线强弱的合法证据。
2. 借真实改动数据收紧机制引用纪律：技能/数值只准引用材料，正面压制机制编造。

## 方案（已选 A：纯 prompt 层）

- `champSelect.ts` 对我方已选英雄 + 敌方已亮英雄并发调 `getChampionPatchNote(championId)`
  （服务自带模块级缓存与网络降级：失败/无改动 → null，不阻塞）。
- 在【敌方情报】之后新增一节：

  ```
  【本版本英雄改动】（数据源：国服更新公告；只列有改动的英雄，未列出=本版本无改动）
  - 敌方疾风剑豪｜削弱：Q - 斩钢闪；冷却 4秒 → 5秒；…
  - 我方赏金猎人｜加强：…
  ```

  双方均无改动时输出固定句：`双方英雄本版本均无官方改动。`
- direction 映射：buff→加强 / nerf→削弱 / adjusted→调整。
- 每英雄 lines 以"；"拼接，上限 8 条，超出截断加"…"（当前公告单英雄普遍 ≤6 条）。
- 纪律修改：原"禁止编造技能/机制/数值"追加唯一例外——【本版本英雄改动】中明确给出的
  技能名与数值可原样引用，禁止改写或外推；改动只能用于说明强弱趋势，不得把旧数值当现状。
- 关键对线模板提示语加入"版本改动"作为可引用维度。

不做：后端批量聚合命令（服务层已有缓存去重）；team.ts 对局中 prompt（下轮再说）。

## 测试

- vitest（champSelect.spec.ts）：mock `@renderer/services/patchNotes`——
  a) 有改动英雄按"我方/敌方 + 方向 + 条目"出现在新节；
  b) 无改动英雄不出现在该节；
  c) 全部无改动 → 固定句；
  d) 纪律含"唯一例外/禁止改写"关键词。
- 验收：复跑多模型 A/B bench（重点看 qwen-max 机制编造是否被真实改动数据替代）。
